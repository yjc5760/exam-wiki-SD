```

  - CLAUDE.md（身份層：分工、資料流、重要規則）

  - raw/json/question_index.json（確認哪些題目已有解析）
不需要手動上傳任何檔案
```

**執行規則：**
```
【階段一：建立資料夾並等待截圖（觸發語句後立即執行）】

1. 讀取 raw/exams/SD-YYYY_結構動力分析與耐震設計.pdf
2. 讀取 raw/json/question_index.json，找出尚無解析的題目

   → mkdir raw/solutions/SD-YYYY-N/（每題一個，已有解析者跳過）

   → 有附圖的題目：列出，提醒使用者截圖並存入對應路徑：
       raw/solutions/SD-YYYY-N/SD-YYYY-N-fig-1.png
   → 無附圖的題目：標注「無需截圖」
5. 輸出截圖清單後停止，等待使用者通知「截圖完成，請開始解題」

7. 依序解析尚無解析的題目，但每次只解一題：
   → 解完一題，存檔後停止
   → 使用者確認後說「繼續下一題」，再解下一題
   → 目的：避免單次輸出超過 32,000 token 上限
8. 每道題目輸出一份獨立的 .md 檔案：
   → 直接存入 raw/solutions/SD-XXXX-N/SD-XXXX-N.md
9. 所有數學公式使用 LaTeX 格式
10. 每份解析開頭必須包含標籤區塊
11. 判斷是否需要輸出互動圖（依「圖形觸發條件」）：
    → 直接存入 raw/solutions/SD-XXXX-N/SD-XXXX-N-[內容碼]-viz.html
```

#### 解題前置：三層掃描法（每題解題前先執行）

**核心原則：** 先見林（外在全貌）→ 再見樹（策略路徑）→ 後見土（內在意圖）

**第一層：外在掃描（2分鐘內完成）**
```
[目標] 這題最終要我交出什麼？（自然頻率？最大位移？設計地震力？隔震週期？）
[已知] 題目給了哪些數字和條件？（質量 m、剛度 k、阻尼比 ξ、地震加速度、規範版本）
[待算] 從已知到目標，需要算出哪些中間參數？（ω、T、Sd、Sa、V、位移反應...）
```

**第二層：策略掃描**
```
[邏輯] 適用哪個分析方法？（見下方「設計哲學框架」）
[陷阱] 考官可能在哪裡挖了坑？（阻尼比影響反應、規範版本差異、SRSS vs CQC、週期上限下限）
[捷徑] 有沒有條件可以跳過某些步驟？（SDOF→不需模態疊加；輕阻尼近似→忽略阻尼自然頻率修正）
```

**第三層：內在掃描**
```
[核心] 這題最重要的一個觀念是什麼？（共振放大、反應譜物理意義、性能設計哲學、隔震延長週期）
[線索] 特殊用字給了什麼暗示？（「有阻尼」→ωd；「設計地震」vs「最大地震」；「橋梁」→用橋梁規範）
```

#### 設計哲學框架（解題邏輯對照）

**SDOF 自由振動**
```
核心：ẍ + 2ξω₀ẋ + ω₀²x = 0
      ω₀ = √(k/m)，ξ = c/(2mω₀)，ωd = ω₀√(1−ξ²)
欠阻尼（ξ < 1）：x(t) = e^(−ξω₀t)[A cos(ωd·t) + B sin(ωd·t)]
初始條件：x(0) = x₀，ẋ(0) = v₀
          A = x₀，B = (v₀ + ξω₀x₀)/ωd
週期：T = 2π/ω₀（無阻尼）；Td = 2π/ωd（有阻尼）
對數衰減率：δ = ln(xn/x_{n+1}) = 2πξ/√(1−ξ²) ≈ 2πξ（輕阻尼）
```

**SDOF 強迫振動（穩態）**
```
核心：mẍ + cẋ + kx = F₀ sin(Ωt)
穩態解：x(t) = X sin(Ωt − φ)
動力放大係數：D = 1/√[(1−r²)² + (2ξr)²]
      r = Ω/ω₀（頻率比）
      φ = arctan[2ξr/(1−r²)]
共振（r ≈ 1）：D_max ≈ 1/(2ξ)（輕阻尼）
最大靜變位：x_st = F₀/k
最大動態位移：X = D·x_st
```

**Duhamel 積分（任意激振）**
```
核心：x(t) = (1/mωd)∫₀ᵗ F(τ)·e^[−ξω₀(t−τ)]·sin[ωd(t−τ)]dτ
地震激振（地盤加速度 üg(t)）：
  相對位移 u(t) = −(1/ωd)∫₀ᵗ üg(τ)·e^[−ξω₀(t−τ)]·sin[ωd(t−τ)]dτ
反應譜：
  Sd(T,ξ) = max|u(t)|        ← 位移反應譜
  Sv(T,ξ) = max|u̇(t)|        ← 速度反應譜（偽速度）
  Sa(T,ξ) = ω₀²·Sd(T,ξ)     ← 加速度反應譜（偽加速度）
  PSv ≈ ω₀·Sd（偽速度）；PSa ≈ ω₀²·Sd（偽加速度）
```

**MDOF 系統自由振動（模態分析）**
```
核心：[M]{ẍ} + [C]{ẋ} + [K]{x} = {0}
特徵值問題：([K] − ω²[M]){φ} = {0}
  → 求解 n 個自然頻率 ω₁ < ω₂ < ... < ωₙ
  → 對應 n 個振態向量 {φ}₁, {φ}₂, ..., {φ}ₙ
正規化：{φ}ᵢᵀ[M]{φ}ⱼ = δᵢⱼ（正交性）
模態參與因子：Γᵢ = {φ}ᵢᵀ[M]{1} / ({φ}ᵢᵀ[M]{φ}ᵢ)
有效質量：mᵢ* = Γᵢ²·mᵢ**；Σmᵢ* = 總質量（驗算）
```

**模態疊加法（反應譜分析）**
```
各模態最大反應：Xᵢ = Γᵢ·Sd(Tᵢ,ξᵢ)·{φ}ᵢ
SRSS 組合：X_total = √[ΣXᵢ²]（各模態頻率相差 > 10%）
CQC 組合：X_total = √[ΣΣρᵢⱼ·Xᵢ·Xⱼ]
  相關係數：ρᵢⱼ = 8ξ²(1+r)r^(3/2) / [(1−r²)² + 4ξ²r(1+r)²]
  r = ωⱼ/ωᵢ
```

**台灣建築耐震設計規範（設計地震力）**
```
核心：設計水平地震力 V = Sa(T)·W / g
反應譜（SMS/SM1 為最大考量地震）：
  設計地震：SDS = (2/3)·SMS，SD1 = (2/3)·SM1
  短週期加速度：SMS = Fa·Ss
  長週期加速度：SM1 = Fv·S1（1秒週期）
  放大係數 Fa、Fv 依場址類別（SA~SF）查表
反應譜形狀：
  T ≤ T₀：Sa = SDS·[0.4 + 0.6T/T₀]
  T₀ ≤ T ≤ Ts：Sa = SDS
  Ts < T ≤ TL：Sa = SD1/T
  T > TL：Sa = SD1·TL/T²
  T₀ = 0.2·Ts，Ts = SD1/SDS
設計地震力：
  V = (SDS/R)·Ie·W（短週期控制）
  最小地震力：V ≥ (SD1·Ie)/(T·R)·W
  最大地震力：V ≤ (SDS·Ie)/(R)·W（T < Ts 時控制）
  極低地震力下限：V ≥ 0.01W
週期計算（建築）：
  T_a = Ct·hₙˣ（經驗公式）
  RC剪力牆：Ct=0.0488，x=0.75
  鋼筋混凝土：Ct=0.0466，x=0.9
```

**橋梁耐震設計規範（α法）**
```
核心：水平地震力 = α·W
設計地震加速度係數 A（依地震分區）
場址類別修正係數 S
等效靜力法：
  P = A·S·C·W
  C = 1.2S/T^(2/3)（慣性力係數）
  最大值：C ≤ 2.5A
  最小值：C ≥ 0.4A·S（長週期）
  橋梁重要性因子 I（生命線 = 1.5；主要 = 1.25；一般 = 1.0）
基礎液化評估：
  剪應力比 τ/σ'v vs 液化阻抗 CRR
  安全係數 FL = CRR/CSR
```

**隔震設計**
```
核心：延長週期，遠離地盤卓越週期；增加阻尼，減小反應
隔震系統等效線性化：
  等效剛度：Keff = F_max / D_max（割線剛度）
  等效阻尼比：βeff = W_D / (2π·Keff·D_max²)
  W_D = 遲滯迴圈面積（消散能量）
隔震週期：TIS = 2π√(W/(g·Keff))
設計位移：DD = g·SD1·TIS / (4π²·BD)
  BD = 阻尼修正係數（查表，βeff 函數）
最大位移（含扭轉）：DM = DD·(1 + y·12e/(b²+d²))
  y = 偏心方向距離，e = 偏心距
隔震層剪力（設計地震）：Vs = Keff·DD
上部結構設計：使用放大係數 Ω₀
```

**消能減震（被動控制）**
```
粘滯阻尼器：F = C·|u̇|^α（α=1線性，α<1非線性）
摩擦阻尼器（BRB）：消散能量 = 降伏力×塑性變形×2
  等效黏滯阻尼比：ξeq = W_D / (4π·Ke·u_max²)
系統等效阻尼：ξtotal = ξstructure + ξadded
  → 反應由修正阻尼反應譜計算
```

### 【QUERY】查詢知識庫並存回（在 Cowork 執行）

**觸發方式：** 直接向 Cowork 提問，問完後說「把這個答案存進知識庫」

**適用情境：**
```
跨題目分析：「SDOF 分析在歷年考題中如何演變？」
弱點整理：「反應譜分析有哪些常見陷阱？」
比較分析：「SRSS 和 CQC 組合的核心差異是什麼？」
趨勢判斷：「隔震設計最近三年考什麼？今年可能考什麼？」
考前衝刺：「哪些三星陷阱出現頻率最高？」
```

**執行流程：**
```
1. Cowork 讀取 wiki/index.md 定位相關分類
2. Cowork 讀取相關 concepts/、problems/、traps/、methods/ 頁面
3. Cowork 合成跨頁面的分析答案
4. 你說「把這個答案存進知識庫」
5. Cowork 存成 wiki/queries/[主題]-[日期].md
```

**wiki/queries/[id].md 格式：**
```markdown
# [查詢主題]

**查詢日期：** YYYY-MM-DD
**關鍵字：** `標籤1` `標籤2`
**參考來源：** [[概念頁]] · [[題目頁]]

## 問題
[你當時問的問題]

## 分析結果
[Cowork 合成的答案]

## 關鍵結論
[2-3 條最重要的洞察]
```
