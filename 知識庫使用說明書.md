## 1. 知識庫架構一覽

知識庫分為**七個知識層**，前三層由 Cowork 透過 compile/ingest 生成，後四層由 Cowork 直接維護：

| 層 | 目錄 | 內容 | 生成方式 |
|----|------|------|---------| 
| Layer 1 | `wiki/concepts/` | 核心概念 | Cowork (compile-all) |
| Layer 1 | `wiki/problems/` | 題解析頁 | Cowork (ingest) |
| Layer 2 | `wiki/philosophy/` | 設計哲學：動力分析/耐震各題型解題邏輯 | Cowork (compile-all) |
| Layer 3 | `wiki/methods/` | 解題方法論：杜哈美積分、模態疊加法、反應譜分析 | Cowork (compile-all) |
| Layer 4 | `wiki/diagnosis/` | 題型診斷決策樹 | Cowork (直接存入) |
| Layer 5 | `wiki/failure-modes/` | 失敗模式：共振/層間位移過大/脆性破壞 | Cowork (直接存入) |
| Layer 6 | `wiki/materials/` | 系統行為：阻尼/勁度/質量 | Cowork (直接存入) |
| Layer 7 | `wiki/code-ref/` | 規範條文對應：建築物耐震設計規範 | Cowork (直接存入) |

**主導航入口：** `wiki/index.md`（七層全覽）、`wiki/by-year.md`（依考年）

## 2. 核心工作流程

```
【解析新考題】
  對 Cowork 說：「解析 XXXX 年考卷」
    ↓ Cowork 掃描 question_index.json，建立題目資料夾
  你把題目附圖截圖（fig-N.png）存入對應資料夾
    ↓
  告知 Cowork：「截圖完成，請開始解題」
    ↓ Cowork 一次解一題，存 .md 檔（含 LaTeX 公式）
  你驗算後對 Cowork 說：「將 SD-XXXX-N 的 verificationStatus 改為 verified」
    ↓
  對 Cowork 說：「ingest SD-XXXX-N」→ wiki 自動更新

【新增概念/方法】
  對 Cowork 說：「add concept [概念名]」或「add method [方法名]」
    ↓ 引導建立 concepts.json 條目 + wiki 頁面

【複習查詢】
  直接向 Cowork 提問，或瀏覽 wiki/ 目錄
```

> **所有操作都對 Cowork 說即可，不需要終端機。**

## 3. Cowork 指令速查

所有指令分三大類，完整說明見 [CLAUDE-CODE.md](CLAUDE-CODE.md)。

### 3.1 解題與 raw 資料維護

| 觸發語句 | 功能 |
|---------|------|
| `解析 XXXX 年考卷` | 掃描全年，建立題目資料夾並提醒截圖 |
| `解析 SD-XXXX-N` | 直接解單題（需已有附圖） |
| `繼續下一題` | 觸發解析同一份考卷的下一題 |
| `SD-XXXX-N 已解析完成` | 同步索引 hasSolution=true 及 tags |
| `將 SD-XXXX-N 的 verificationStatus 改為 verified` | 人工驗算通過後標記 |
| `將 SD-XXXX-N 的 verificationStatus 改為 needs-review` | 發現錯誤時標記 |

### 3.2 wiki 編譯與維護（7個）

| 觸發語句 | 功能 | 前置條件 |
|---------|------|---------|
| `ingest SD-XXXX-N` | 將一道題目寫入 wiki | verificationStatus = "verified" |
| `compile all` | 從 raw/ 重建整個 wiki | — |
| `lint wiki` | 健檢 wiki 完整性（16 項） | — |
| `status` | 驗證進度統計 | — |
| `reindex` | 修正 hasSolution 不一致 | — |
| `add concept [名稱]` | 新增概念到 concepts.json | — |
| `add method [名稱]` | 新增解題方法論頁面 | — |

### 3.3 備考分析類（4個）

| 觸發語句 | 功能 |
|---------|------|
| `frequency` | 統計各考點歷年出現次數，找出高頻考點 |
| `analyze YYYY` | 深度分析某一年考卷的考點分布 |
| `predict` | 根據歷年頻率推測最可能出現的考點 |
| `study SD-UN` | 彙整某單元所有題目、重點公式、常見陷阱 |

### 3.4 查詢快捷類（4個）

| 觸發語句 | 功能 |
|---------|------|
| `find [關鍵字]` | 搜尋含某關鍵字的題目 |
| `related SD-XXXX-N` | 找出與某題考點相似的其他題目 |
| `unverified` | 列出所有有解析但尚未驗算的題目 |
| 直接提問 | 自由查詢知識庫 |

## 4. 複習使用指南

### 4.1 依考年複習

開啟 `wiki/by-year.md` → 選年份 → 點入題目頁 → 查看核心考點與解題步驟。

### 4.2 依題型複習（診斷決策樹）

| 題型 | 診斷頁 |
|------|--------|
| SDOF (單自由度系統) | `wiki/diagnosis/sdof.md` |
| MDOF (多自由度系統) | `wiki/diagnosis/mdof.md` |
| 反應譜分析 | `wiki/diagnosis/spectrum.md` |
| 規範耐震設計 | `wiki/diagnosis/code-design.md` |

### 4.3 查詢規範數字的來源（code-ref/）

開啟 `wiki/code-ref/index.md` 查詢建築物耐震設計規範條文對應。

### 4.4 術語速查

開啟 `wiki/concepts/` 查詢個別概念說明。

### 4.5 直接問 Cowork

```
單自由度系統阻尼比如何計算？
模態疊加法怎麼使用？
SD-2018-3 的考點是什麼？
哪幾題考過反應譜分析？
剛度矩陣和質量矩陣如何建立？
```

## 5. 常見問題

**Q：解析 .md 放在哪裡？**
- 原始來源（唯讀）：`raw/solutions/SD-XXXX-N/SD-XXXX-N.md`
- 知識庫版本（ingest 後）：`wiki/problems/SD-XXXX-N.md`

**Q：如何更正已 verified 的答案？**
對 Cowork 說：「將 SD-XXXX-N 的 verificationStatus 改為 needs-review」→ 說明錯誤 → 重新驗算 → 改回 verified → 說「ingest SD-XXXX-N」。

**Q：新增的概念頁 / 方法頁需要 ingest 嗎？**
不需要。`wiki/concepts/`、`wiki/methods/`、`wiki/code-ref/` 等由 Cowork 直接維護，不走 ingest。

**Q：截圖要怎麼命名？**

| 類型碼 | 內容 | 範例 |
|--------|------|------|
| `fig` | 題目附圖 | `SD-2018-1-fig-1.png` |
| `chart` | 設計圖表 | `SD-2018-1-chart-1.png` |
| `eqn` | 公式截圖 | `SD-2018-1-eqn-1.png` |
| `hand` | 手寫補充 | `SD-2018-1-hand-1.png` |

**Q：目前解析進度？**

對 Cowork 說 `status` 即可取得最新統計。目前概況：
- 解析資料夾：涵蓋 2002–2025 年的結構動力分析與耐震設計考題
- 狀態請執行 `status` 動態確認
